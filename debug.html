省略変換器 Debug Site

Chrome拡張による日本語変換の挙動確認用ページ

1. 固定置換テスト

rules.json5 の literal 系ルールが効くか確認します。

それをやることにした。
やっぱりそれは分かる。
それについて書き出す。

2. 動詞・連用形テスト

「分かる」「当たる」「悩み」「書き出す」などの変換確認用。

文章を変換したい。
意味が分かる文章にしたい。
面倒ごとに当たる。
悩みを書き出す。

3. 「こと」単独トークン変換テスト

単独トークンとして解析される「こと」が一律で「ヿ」に変換されることを確認します。

日記を書くことにした。
変換することができる。
大事なことを忘れた。
そのことについて考える。

4. 句読点・記号テスト

将来的に「、→､」「。→｡」などを入れた時の確認欄。

これは、テストです。さらに、確認します。
9時から14時半まで作業した。

5. 動的DOM追加テスト

ボタンを押して後から追加された文章にも、自動で変換が適用されることを確認します。

6. textarea入力欄

通常のcontent scriptは、text nodeは変換しますが、textareaのvalueは別処理が必要です。

7. 確認ポイント

  • ページ表示後、本文中の「それ」が「其」へ変換されるか。
  • 「やっぱり」が「矢ッ張」へ変換されるか。
  • 「分かる」が「分る」へ変換されるか。
  • 単独トークンとして解析された「こと」が「ヿ」へ変換されるか。
  • 「ことば」のような別トークン語が巻き込まれないか。
  • 後から追加した文章にも自動で変換が反映されるか。
  • Console に 変換初期化完了 と MutationObserver 開始ログが出るか。
