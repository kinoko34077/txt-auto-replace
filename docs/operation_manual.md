# 日本語表記変換システム 運用指示書 (v1.3)

この文書は、日本語表記変換システム（Chrome 拡張機能）の利用方法、設定手順、運用上の注意を示す運用指示書である。プログラムの構造や設計思想は別途「プログラム仕様書」にまとめてあるため、ここでは利用者がシステムをインストールし、カスタマイズし、日常的に運用するための実践的な手順を提供する。

## 1. インストール手順

1. **ファイル配置** – システムは `jpn-transform-ext` というフォルダに格納されている。フォルダ内には `manifest.json`、`content.js`、`transform-bundles.json5`、`transforms/`、`lib/`、`dict/` などのファイルが含まれている。これらをまとめた ZIP アーカイブから取り出し、任意のディレクトリに配置する。

2. **Chrome 拡張機能の読み込み** – Chrome ブラウザを開き、アドレスバーに `chrome://extensions` と入力して拡張機能管理画面を開く。右上の「デベロッパーモード」をオンにし、「パッケージ化されていない拡張機能を読み込む」ボタンを押し、先ほどの `jpn-transform-ext` フォルダを選択する。読み込みが成功すると拡張が有効化され、ツールバーにアイコンが表示される。

3. **辞書ファイルの確認** – 形態素解析には `dict/` フォルダ内の辞書ファイルが必要である。`dict/` に Kuromoji の辞書ファイル一式（`.dat` や `.def` など）が含まれていることを確認する。辞書ファイルが不足していると解析が失敗するので注意する。

4. **デバッグ環境の準備** – 変換結果を確認するために、システムには簡易デバッグページ（例: `debug.html`）を用意することが推奨される。これは標準 HTML ページで、固定テキストやボタンが含まれており、拡張がテキストノードを書き換える様子を視覚的に確認できる。`start_localhost.bat` を実行すると、リポジトリ直下を `http://127.0.0.1:8000/` で配信できる。

## 2. 基本的な使い方

1. **拡張の動作確認** – 拡張機能を有効にした状態で任意の Web ページを開く。ページ中の日本語テキストが自動で変換される。デバッグページでは、特定の文例に対して期待する変換が行われているかを確認する。例として、

   - 「それをやることにした。」→「其をやるヿにした。」
   - 「やっぱりそういうことか。」→「矢ッ張そういうことか。」
   - 「文章を変換したい。」→「文章を変換したい。」（誤変換がないことを確認）

2. **Console ログの利用** – Chrome の DevTools を開き (`F12` または `Ctrl+Shift+I`)、Console タブを確認すると、変換規則の適用状況やエラーが出力される。規則に一致したトークンやスキップされたトークンが確認できるため、規則を追加する際のデバッグに役立つ。

3. **ページの更新と再変換** – 拡張はページロード時に初回変換を行い、その後も MutationObserver により動的に生成されるコンテンツ（例えば Ajax で後から挿入される文章）を監視して再変換する。動的ページでは、追加された文章が自動で変換されることを確認する。

## 3. 変換規則のカスタマイズ

### 3.1 変換バンドルの構造

変換規則は 1 ファイルに集約せず、`transform-bundles.json5` と `transforms/*.json5` に分けて管理する。`transform-bundles.json5` では適用順と有効状態を定義し、各バンドルファイルには実際の規則を置く。バンドル形式は、通常の `token-rules` と、固定辞書向けの `dictionary-rules` を使い分ける。

```js
{
  bundles: [
    {
      id: "legacy-kanji",
      label: "旧字変換",
      path: "transforms/40-legacy-kanji.json5",
      enabled: true,
      order: 40,
    },
    {
      id: "general-character-replacements",
      label: "一般単漢字置換",
      path: "transforms/60-general-character-replacements.json5",
      enabled: true,
      order: 60,
    },
  ],
}
```

各バンドル内の規則の基本形式は次の通りである。

```js
{
  from: "変換前文字列",        // トークンの表層形
  to: "変換後文字列",          // 置換後の表記
  priority: 100,               // 優先順位（大きいほど先に適用）
  conditions: {
    prev: [ { pos: "動詞", cform: "基本形" } ],
    current: [ { pos: "名詞", basic: "こと" } ],
    next: [ { pos1: "格助詞" } ]
  }
}
```

複合語や周辺語で意味が決まるものは、単一トークン規則ではなく `sequence` を使って記述できる。

```js
{
  sequence: [
    { surface: "面倒", pos: "名詞" },
    { surface: "ごと", pos: "名詞" },
  ],
  to: "面倒事",
  priority: 90,
}
```

旧字変換や同音漢字置換のように、固定熟語と単漢字を分けて積みたい場合は `dictionary-rules` を使う。

```js
{
  id: "legacy-kanji",
  label: "旧字変換",
  kind: "dictionary-rules",
  phrase_rules: [
    { from: "総合", to: "綜合", priority: 80 }
  ],
  character_map_priority: 10,
  character_map: {
    "両": "兩",
    "乗": "乘",
  }
}
```

- **`from`** – 対象となるトークンの `surface_form`。
- **`to`** – 変換後の文字列。空文字にすることで削除も可能。
- **`priority`** – 規則の適用順序。数値が大きいほど先に評価される。
- **`conditions`** – 任意フィールド。`prev`/`current`/`next` の各キーには条件の配列を指定する。配列内のオブジェクトは AND 条件、配列間は OR 条件として評価される。使用可能なフィールドは `pos`, `pos_detail_1`（`pos1`）、`conjugated_form`（`cform`）、`conjugated_type`（`ctype`）、`surface`、`basic` など。

### 3.2 設定ページの使い方

拡張機能の詳細画面から「拡張機能のオプション」を開くと、`legacy-kanji`、`general-character-replacements`、`homophone-kanji` の 3 箱を個別編集できる。ここで保存した内容は `chrome.storage.local` に入り、そのブラウザ上でバンドル定義ファイルより優先される。

1. **箱を選ぶ** – 旧字参照表に含まれる項目は `legacy-kanji`、参照表外の単漢字置換は `general-character-replacements`、同音熟語は `homophone-kanji` を使う。
2. **項目を編集する** – 単漢字置換と固定熟語置換を別々に編集できる。危険な項目は単漢字で持たず、熟語や条件付きルールへ逃がす。
3. **保存する** – 「この箱を保存」または「全体を保存」を押す。
4. **ページを再読み込みする** – content script は初期化時に規則を読むため、保存後は変換対象のタブを再読み込みする。

### 3.3 規則追加の手順

1. **バンドルを選ぶ** – 追加したい規則がどの箱に属するかを決める。固定の旧字や同音熟語であれば専用バンドルに入れ、汎用的な語彙置換や送り仮名省略なら既存の汎用バンドルへ入れる。

2. **新しい規則を定義** – 対応する `transforms/*.json5` をテキストエディタで開き、既存規則の末尾にカンマ区切りで新しい規則オブジェクトを追加する。JSON5 は末尾カンマを許容するため、追加が容易である。

3. **コメントで意図を記述** – どのような目的で規則を追加したか、参考にした資料や例文をコメントとして書いておくと、将来の保守に役立つ。

4. **条件判定を設計** – 誤変換を避けるため、必要に応じて `prev` や `next` 条件を設定する。例えば動詞系の語形変換では前後トークン条件を使い、複合語や固定連接は `sequence` で表現する。

5. **順序が必要ならバンドル定義を更新** – 新しい箱を増やす場合は `transform-bundles.json5` に追加し、`order` を調整する。

6. **拡張を再読み込み** – バンドル定義や `transforms/*.json5` を保存した後、拡張機能管理画面で拡張を再読み込みし、対象ページも再読み込みすると新しい規則が反映される。

### 3.3 規則の削除・無効化

規則を無効にしたい場合は、該当する `transforms/*.json5` からオブジェクトを削除するか、`enabled: false` を付ける。バンドル全体を外したい場合は `transform-bundles.json5` 側でそのバンドルを `enabled: false` にする。

### 3.4 注意事項

- JSON5 は厳密な JSON と異なりコメントと末尾カンマを許容するが、誤ってシングルクォートとダブルクォートを混在させないよう注意する。
- 形態素解析に依存するため、未知語や固有名詞が辞書に含まれていない場合は `pos` が `名詞, 固有名詞` 等となる。複雑な語彙には過剰に規則を設けず、誤変換を避ける工夫が必要。
- Stage5（旧字変換）は、ユーザー指定の旧字参照表だけを `legacy-kanji` へ入れている。そこに含めない単漢字置換は `general-character-replacements` へ逃がす。
- 同音漢字置換は現時点では `奇跡 → 奇蹟` を実装済みで、今後も基本は固定熟語単位または単漢字単位で管理する。誤爆しやすいものだけを条件付きルールとして別管理する。
- Stage6（ルビ処理）は未実装である。これに対応する規則を書いても現行バージョンでは機能しないため、将来のアップデートまで待つ。

## 4. トラブルシューティング

1. **変換が適用されない** – `transform-bundles.json5` や `transforms/*.json5` の記述に誤りがないか確認する。特に条件のキー名 (`pos`/`cform` など) の綴りや配列のカンマ位置に注意する。また、`priority` や `order` の設定次第で、意図した順序で規則が走らない可能性がある。

2. **誤変換が発生する** – 条件を厳密に指定していない場合、予期せぬ文脈で規則が適用されることがある。`prev` や `next` の条件を追加し、トークンの `pos` や `conjugated_form` を絞り込むことで誤変換を抑えられる。デバッグ時は Console に各トークンの属性を出力すると良い。

3. **ページが崩れる・非表示になる要素がある** – 変換はテキストノードの内容のみを書き換えるため、HTML 構造には影響しないはずだが、script タグ内や属性値まで書き換えるとページ動作に影響が出る。`content.js` ではテキストノードのみを処理していることを確認し、コードやスタイルの一部が書き換えられていないかチェックする。

4. **形態素解析のエラー** – `dict/` 内の辞書ファイルが欠落していると `kuromoji.builder()` がエラーを返す。辞書ファイルを正しいパスに配置しているか、`dicPath` の指定が正しいかを確認する。辞書一式は約1.2MBと比較的大きいため、欠けていないかチェックする。

## 5. 保守・更新

- **バージョン管理** – `manifest.json` のバージョン (`version`) を更新することで Chrome がアップデートを認識する。規則追加や機能改善を行った際はバージョン番号を増やして管理する。

- **拡張のバックアップ** – `transform-bundles.json5` や `transforms/*.json5` をカスタマイズした場合はバックアップを取り、Git などのバージョン管理システムで変更履歴を追跡することが推奨される。特に誤って規則を削除してしまった場合に復元しやすい。

- **辞書更新** – Kuromoji の辞書は時折更新される。未知語や新語に対応したい場合は、開発者が辞書ファイルを更新し、`dict/` フォルダを置き換えることで対応可能である。

- **拡張機能の再パッケージ** – 機能を追加したらフォルダ全体を ZIP に再パッケージし、ユーザに配布する。Chrome Web Store で配布する場合は審査を経る必要がある。

## 6. FAQ（よくある質問）

### Q1: `index.html` を直接開くと拡張が動作しないのはなぜですか？

A1: `file://` スキームでは fetch() や外部リソースの読み込みが制限されるため、`transform-bundles.json5`、`transforms/*.json5`、`dict/` が読み込めず解析に失敗します。デバッグ時は `start_localhost.bat` を実行して `http://127.0.0.1:8000/index.html` または `http://127.0.0.1:8000/debug.html` を開いてください。VSCode の Live Server や `python -m http.server` でも代用できます。

### Q2: JSON5 でコメントを入れるのは安全ですか？

A2: はい。JSON5 はコメント (`//` や `/* ... */`) を許容しますが、パーサーが JSON5 形式に対応している必要があります。本システムでは `json5.min.js` を読み込んでいるため、`transform-bundles.json5` と `transforms/*.json5` は自由にコメントを含められます。 YAML や他の形式は Chrome 拡張機能では読み込みが困難なため採用していません。

### Q3: `こと` を `ヿ` に変換したいが、「ことば」や「まこと」を誤変換したくありません。

A3: Kuromoji は形態素単位でトークンを生成するため、「こと」は単独トークンとして処理され、「ことば」は別のトークンになります。対応する `transforms/*.json5` で `from: "こと"` と記述すれば「ことば」は一致しません。また、前後の条件 (`prev` に動詞の基本形を指定する等) を追加して変換を限定することも可能です。

### Q4: 拡張機能を別のブラウザでも使えますか？

A4: 本システムは Chrome 拡張 v3 を前提としているため、Chromium ベースの Edge などでも動作する可能性があります。ただしブラウザ固有の API 差異や権限設定に注意してください。Firefox 用アドオンへの移植には `manifest.json` や API 呼び出しを調整する必要があります。

## 7. まとめ

本運用指示書では、日本語表記変換システムのインストール方法、基本的な使い方、変換規則の編集方法、トラブルシューティング、保守・更新手順を説明した。プログラム仕様書で記述された設計を踏まえ、ユーザが安全かつ効果的に本システムを運用できるよう、具体的な手順と注意事項を整理している。運用者は定期的に規則を見直し、必要に応じて辞書や拡張機能本体を更新することで、変換精度と安定性を維持できる。
