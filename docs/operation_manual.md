# 日本語表記変換システム 運用指示書 (v1.1)

この文書は、日本語表記変換システム（Chrome 拡張機能）の利用方法、設定手順、運用上の注意を示す運用指示書である。プログラムの構造や設計思想は別途「プログラム仕様書」にまとめてあるため、ここでは利用者がシステムをインストールし、カスタマイズし、日常的に運用するための実践的な手順を提供する。

## 1. インストール手順

1. **ファイル配置** – システムは `jpn-transform-ext` というフォルダに格納されている。フォルダ内には `manifest.json`、`content.js`、`rules.json5`、`lib/`、`dict/` などのファイルが含まれている。これらをまとめた ZIP アーカイブから取り出し、任意のディレクトリに配置する。

2. **Chrome 拡張機能の読み込み** – Chrome ブラウザを開き、アドレスバーに `chrome://extensions` と入力して拡張機能管理画面を開く。右上の「デベロッパーモード」をオンにし、「パッケージ化されていない拡張機能を読み込む」ボタンを押し、先ほどの `jpn-transform-ext` フォルダを選択する。読み込みが成功すると拡張が有効化され、ツールバーにアイコンが表示される。

3. **辞書ファイルの確認** – 形態素解析には `dict/` フォルダ内の辞書ファイルが必要である。`dict/` に Kuromoji の辞書ファイル一式（`.dat` や `.def` など）が含まれていることを確認する。辞書ファイルが不足していると解析が失敗するので注意する。

4. **デバッグ環境の準備** – 変換結果を確認するために、システムには簡易デバッグページ（例: `debug.html`）を用意することが推奨される。これは標準 HTML ページで、固定テキストやボタンが含まれており、拡張がテキストノードを書き換える様子を視覚的に確認できる。

## 2. 基本的な使い方

1. **拡張の動作確認** – 拡張機能を有効にした状態で任意の Web ページを開く。ページ中の日本語テキストが自動で変換される。デバッグページでは、特定の文例に対して期待する変換が行われているかを確認する。例として、

   - 「それをやることにした。」→「其をやるヿにした。」
   - 「やっぱりそういうことか。」→「矢ッ張そういうことか。」
   - 「文章を変換したい。」→「文章を変換したい。」（誤変換がないことを確認）

2. **Console ログの利用** – Chrome の DevTools を開き (`F12` または `Ctrl+Shift+I`)、Console タブを確認すると、変換規則の適用状況やエラーが出力される。規則に一致したトークンやスキップされたトークンが確認できるため、規則を追加する際のデバッグに役立つ。

3. **ページの更新と再変換** – 拡張はページロード時に初回変換を行い、その後も MutationObserver により動的に生成されるコンテンツ（例えば Ajax で後から挿入される文章）を監視して再変換する。動的ページでは、追加された文章が自動で変換されることを確認する。

## 3. 変換規則のカスタマイズ

### 3.1 `rules.json5` の構造

変換規則は `jpn-transform-ext/rules.json5` に JSON5 形式で記述する。JSON5 はコメントや末尾カンマを許容するため、各規則の意図や参照元をコメントとして記述できる。規則の基本形式は次の通りである。

```js
{
  from: "変換前文字列",        // トークンの表層形
  to: "変換後文字列",          // 置換後の表記
  priority: 100,               // 優先順位（大きいほど先に適用）
  conditions: {
    prev: [ { pos: "動詞", cform: "基本形" } ],
    current: [ { pos: "名詞", basic: "こと" } ],
    next: [ { pos1: "格助詞" } ]
  }
}
```

- **`from`** – 対象となるトークンの `surface_form`。
- **`to`** – 変換後の文字列。空文字にすることで削除も可能。
- **`priority`** – 規則の適用順序。数値が大きいほど先に評価される。
- **`conditions`** – 任意フィールド。`prev`/`current`/`next` の各キーには条件の配列を指定する。配列内のオブジェクトは AND 条件、配列間は OR 条件として評価される。使用可能なフィールドは `pos`, `pos_detail_1`（`pos1`）、`conjugated_form`（`cform`）、`conjugated_type`（`ctype`）、`surface`、`basic` など。

### 3.2 規則追加の手順

1. **新しい規則を定義** – `rules.json5` をテキストエディタで開き、既存規則の末尾にカンマ区切りで新しい規則オブジェクトを追加する。JSON5 は末尾カンマを許容するため、追加が容易である。

2. **コメントで意図を記述** – どのような目的で規則を追加したか、参考にした資料や例文をコメントとして書いておくと、将来の保守に役立つ。

3. **条件判定を設計** – 誤変換を避けるため、必要に応じて `prev` や `next` 条件を設定する。例えば「こと」を特定文脈だけで「ヿ」にしたい場合、前が動詞の基本形である場合に限定するなどの設計が可能である。

4. **拡張を再読み込み** – `rules.json5` を保存した後、拡張機能管理画面で拡張を一度無効にしてから再度有効にする（またはページをリロードする）と、新しい規則が反映される。

### 3.3 規則の削除・無効化

規則を無効にしたい場合は、`rules.json5` から該当オブジェクトを削除するか、`priority` を極端に低くする。コメントアウトは JSON5 でサポートされるため、`//` や `/* */` を用いて一時的にコメントアウトすることも可能である。

### 3.4 注意事項

- JSON5 は厳密な JSON と異なりコメントと末尾カンマを許容するが、誤ってシングルクォートとダブルクォートを混在させないよう注意する。
- 形態素解析に依存するため、未知語や固有名詞が辞書に含まれていない場合は `pos` が `名詞, 固有名詞` 等となる。複雑な語彙には過剰に規則を設けず、誤変換を避ける工夫が必要。
- Stage5（旧字変換）や Stage6（ルビ処理）は未実装である。これらに対応する規則を書いても現行バージョンでは機能しないため、将来のアップデートまで待つ。

## 4. トラブルシューティング

1. **変換が適用されない** – `rules.json5` の記述に誤りがないか確認する。特に条件のキー名 (`pos`/`cform` など) の綴りや配列のカンマ位置に注意する。また、`priority` が極端に低いと、他の規則に上書きされる可能性がある。

2. **誤変換が発生する** – 条件を厳密に指定していない場合、予期せぬ文脈で規則が適用されることがある。`prev` や `next` の条件を追加し、トークンの `pos` や `conjugated_form` を絞り込むことで誤変換を抑えられる。デバッグ時は Console に各トークンの属性を出力すると良い。

3. **ページが崩れる・非表示になる要素がある** – 変換はテキストノードの内容のみを書き換えるため、HTML 構造には影響しないはずだが、script タグ内や属性値まで書き換えるとページ動作に影響が出る。`content.js` ではテキストノードのみを処理していることを確認し、コードやスタイルの一部が書き換えられていないかチェックする。

4. **形態素解析のエラー** – `dict/` 内の辞書ファイルが欠落していると `kuromoji.builder()` がエラーを返す。辞書ファイルを正しいパスに配置しているか、`dicPath` の指定が正しいかを確認する。辞書一式は約1.2MBと比較的大きいため、欠けていないかチェックする。

## 5. 保守・更新

- **バージョン管理** – `manifest.json` のバージョン (`version`) を更新することで Chrome がアップデートを認識する。規則追加や機能改善を行った際はバージョン番号を増やして管理する。

- **拡張のバックアップ** – `rules.json5` をカスタマイズした場合はバックアップを取り、Git などのバージョン管理システムで変更履歴を追跡することが推奨される。特に誤って規則を削除してしまった場合に復元しやすい。

- **辞書更新** – Kuromoji の辞書は時折更新される。未知語や新語に対応したい場合は、開発者が辞書ファイルを更新し、`dict/` フォルダを置き換えることで対応可能である。

- **拡張機能の再パッケージ** – 機能を追加したらフォルダ全体を ZIP に再パッケージし、ユーザに配布する。Chrome Web Store で配布する場合は審査を経る必要がある。

## 6. FAQ（よくある質問）

### Q1: `index.html` を直接開くと拡張が動作しないのはなぜですか？

A1: `file://` スキームでは fetch() や外部リソースの読み込みが制限されるため、`rules.json5` や `dict/` が読み込めず解析に失敗します。デバッグ時は VSCode の Live Server や Python の簡易 HTTP サーバー (`python -m http.server`) などで `http://localhost/...` 経由で開いてください。

### Q2: JSON5 でコメントを入れるのは安全ですか？

A2: はい。JSON5 はコメント (`//` や `/* ... */`) を許容しますが、パーサーが JSON5 形式に対応している必要があります。本システムでは `json5.min.js` を読み込んでいるため、`rules.json5` は自由にコメントを含められます。 YAML や他の形式は Chrome 拡張機能では読み込みが困難なため採用していません。

### Q3: `こと` を `ヿ` に変換したいが、「ことば」や「まこと」を誤変換したくありません。

A3: Kuromoji は形態素単位でトークンを生成するため、「こと」は単独トークンとして処理され、「ことば」は別のトークンになります。`rules.json5` で `from: "こと"` と記述すれば「ことば」は一致しません。また、前後の条件 (`prev` に動詞の基本形を指定する等) を追加して変換を限定することも可能です。

### Q4: 拡張機能を別のブラウザでも使えますか？

A4: 本システムは Chrome 拡張 v3 を前提としているため、Chromium ベースの Edge などでも動作する可能性があります。ただしブラウザ固有の API 差異や権限設定に注意してください。Firefox 用アドオンへの移植には `manifest.json` や API 呼び出しを調整する必要があります。

## 7. まとめ

本運用指示書では、日本語表記変換システムのインストール方法、基本的な使い方、変換規則の編集方法、トラブルシューティング、保守・更新手順を説明した。プログラム仕様書で記述された設計を踏まえ、ユーザが安全かつ効果的に本システムを運用できるよう、具体的な手順と注意事項を整理している。運用者は定期的に規則を見直し、必要に応じて辞書や拡張機能本体を更新することで、変換精度と安定性を維持できる。
